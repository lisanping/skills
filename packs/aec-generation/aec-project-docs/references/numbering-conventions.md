# 文档编号规则

> **何时读取此文件**：当需要为 RFI、变更单、联系单、纪要、日报等生成编号时；或解析既有文档编号、推断编号规则时。

## 一、为什么编号重要

项目通信文档的编号是**档案检索和合同争议**的关键索引。编号不规范带来三个直接问题：

1. **检索困难**——半年后想找"那个关于人防口部的 RFI"，没编号根本翻不到
2. **结算扯皮**——变更单缺编号或编号重复，结算阶段难以确认有效性
3. **责任不清**——多份同类文件互相引用，编号混乱时无法判断哪份是最新版

因此 SKILL 在生成任何项目文档时，**必须给出编号**，即使是用户没要求。

---

## 二、SKILL 的编号决策流程

```
┌──────────────────────────────┐
│ 用户是否提供编号规则？         │
└──┬───────────────────────────┘
   │
   ├─ 是 ──→ 严格使用用户规则
   │
   └─ 否
      │
      ┌──────────────────────────────┐
      │ 用户是否提供项目既有文档样例？ │
      └──┬───────────────────────────┘
         │
         ├─ 是 ──→ 从样例反推规则，复用
         │
         └─ 否 ──→ 使用通用回退规则（见下方第 3 节）
                  并明确告知用户："我用了这套回退规则，
                  如果项目有自己的规则请告诉我，可以替换。"
```

**SKILL 不要偷偷使用一套规则不告知用户。** 编号规则在不同公司、不同项目差异很大，未经确认的编号一旦在文档里固化，后续修改成本很高。

---

## 三、通用回退规则

### 标准格式

```
{项目代号}-{文档类型代号}-{年月}-{流水号}
```

例：`SHCS-RFI-202510-018`

### 各字段说明

#### 项目代号

由项目名称缩写而来，**全大写、字母**，2–6 个字符。

| 项目类型 | 代号示例 |
|---------|---------|
| 上海中心 | `SHC` 或 `SHCS` |
| 万达广场（合肥） | `WD-HF` 或 `WDHF` |
| 某机场 T3 航站楼 | `T3` 或 `APT3` |

如用户没说，SKILL 应**问一句**："项目代号用什么？我建议用 [基于项目名称推导的代号]"，让用户确认或替换。

#### 文档类型代号

| 类型 | 代号 | 全称 |
|------|------|------|
| 信息请求单 | `RFI` | Request For Information |
| 设计变更单 | `DC` | Design Change |
| 工程变更单（业主侧） | `CO` | Change Order |
| 工程联系单 | `EC` | Engineering Correspondence |
| 工作联系单（项目内） | `WC` | Work Correspondence |
| 提交物 / 资料报审 | `SUB` | Submittal |
| 监理通知单 | `MN` | Monitor Notice |
| 监理回复单 | `MR` | Monitor Reply |
| 现场签证 | `SQ` | Site Visa（中文"签证"拼音首字母） |
| 设计例会纪要 | `MM-D` | Meeting Minutes - Design |
| 工地例会纪要 | `MM-S` | Meeting Minutes - Site |
| 专题会议纪要 | `MM-T` | Meeting Minutes - Topic |
| 施工日报 | `DR` | Daily Report |
| 施工周报 | `WR` | Weekly Report |
| 监理日志 | `ML` | Monitor Log |
| 检验批质量验收 | `QA` | Quality Acceptance |

> 这套代号是行业较通用的形式，但不是唯一标准。如项目自有代号体系，以项目为准。

#### 年月

`YYYYMM` 六位数字。**不要省略前导零**：2025 年 3 月写 `202503` 不是 `20253`。

部分单位规则不含年月，仅按整年流水（如 `RFI-2025-0123`），SKILL 应识别这种变体。

#### 流水号

按文档类型在年月内独立计数，**3 位数字**起步，不够补零。

- 第 1 份：`001`
- 第 18 份：`018`
- 第 156 份：`156`
- 超过 999 份：扩展为 4 位 `1024`，但极少出现

**关键问题**：SKILL 通常不知道当前流水号已到几号。处理方式：

1. 用户告知 → 按用户提供的下一号
2. 用户没告知 → SKILL 用占位符 `XXX`，并说明"流水号请按项目台账填入"
3. **绝对不要** 默认填 `001`——这会和项目实际编号撞车

## 四、其他常见编号变体

### 变体 1：含合同段编号

大型项目分多个合同段时，编号嵌入合同段：

```
{项目代号}-{合同段}-{文档类型}-{年月}-{流水号}
例：SHCS-C2-RFI-202510-018
```

### 变体 2：含专业代号

设计变更单常嵌入专业：

```
{项目代号}-DC-{专业代号}-{年月}-{流水号}
例：SHCS-DC-M-202510-007  （M = 暖通 Mechanical）
```

常见专业代号：

| 专业 | 代号 |
|------|------|
| 建筑 | `A` (Architecture) |
| 结构 | `S` (Structure) |
| 给排水 | `P` (Plumbing) |
| 电气 | `E` (Electrical) |
| 暖通 | `M` (Mechanical) |
| 智能化 | `IT` 或 `T` |
| 室外总图 | `G` (General) |
| 景观 | `L` (Landscape) |
| 室内 | `ID` (Interior Design) |

### 变体 3：日期型编号（仅用于日报、监理日志）

```
{项目代号}-{文档类型}-{YYYYMMDD}
例：SHCS-DR-20251018
```

日报按日期一一对应，无需流水号。

如同日多份（不同填报方），追加分包代号：

```
SHCS-DR-20251018-八局
SHCS-DR-20251018-暖通
```

### 变体 4：央企体系简写

部分央企总包内部使用更短的格式，省略项目代号（因为系统自带项目归属）：

```
{文档类型}-{年}{月}{日}{流水号}
例：RFI-2510180012  （含义：25 年 10 月 18 日第 12 份 RFI）
```

SKILL 识别这种格式时不要按通用格式重写——以项目实际格式为准。

### 变体 5：外资 / 国际项目格式

外资项目常用英文 + 较长格式：

```
{Project}-{Phase}-{Discipline}-{DocType}-{Sequence}
例：SHCS-CD-MEP-RFI-0123
```

CD = Construction Documents，MEP = Mechanical/Electrical/Plumbing。

如项目是国际背景（甲方、设计、咨询有外资方），优先用英文格式。

## 五、版次（Revision）规则

文档**升版**时不变更主编号，只追加版次：

```
{编号} Rev.{版次}
例：SHCS-DC-202510-007 Rev.A → Rev.B → Rev.C
```

版次约定：

- 首版：通常为 `Rev.0` 或不标版次（视项目惯例）
- 修订后：`Rev.A`、`Rev.B`、`Rev.C`……
- 字母达到 `Rev.Z` 后通常重启大版（极少见）

部分项目用数字：`Rev.1`、`Rev.2`，含义相同。

**注意**：纪要、日报这类**事件型文档不升版**——一旦发出即固定。如内容有错误，应另发**勘误**或**补充**文件，引用原编号但不替换。

## 六、台账登记建议

每生成一份文档，建议同步在项目台账中登记一行。SKILL 在生成主文档后，**可主动建议用户**将下列字段添加到台账：

| 编号 | 类型 | 主送单位 | 简要事由 | 签发日期 | 期限 / 应答日期 | 状态 |
|------|------|---------|---------|---------|---------------|------|
| | | | | | | |

如果用户的台账已知格式（曾在对话中提及），按既有格式输出粘贴行；否则按上表通用格式。

## 七、常见错误

1. **流水号默认 001**——前面已强调，会和实际撞号。
2. **省略年月前导零**（`20253` 当 `202503` 用）——排序和检索都会出错。
3. **类型代号自创且未告知用户**——比如把"工程联系单"写成 `GLD`（拼音首字母），跟通用 `EC` 不一致。SKILL 用任何非通用代号都应当先告知用户。
4. **专业代号混用中英文**——`SHCS-DC-暖通-202510-007` 看起来"接地气"，但与系统其他编号不一致。统一用英文代号。
5. **版次写在主编号中间而非末尾**——`SHCS-DC-RevB-202510-007` 是错的，正确是 `SHCS-DC-202510-007 Rev.B`。
6. **跨年时年月归属判断错误**——12 月 31 日发起的 RFI 应归 12 月，不是次年 1 月。**以签发日期为准**。
7. **编号一旦发出就修改**——已发出的文档编号即使发现规则有问题也不能改，会破坏台账连续性。要么沿用，要么作废重发新编号（在原编号后追加"作废"标记，并发新文）。
