# 设计变更单模板

> **何时读取此文件**：当需要生成设计变更单（Design Change Order）、工程变更通知、变更指令时。

## 一、文档定位

设计变更单是设计单位发起或确认的图纸/方案变更的正式文件。它的法律和合同意义重大：

- 影响**结算**：变更引起的工程量增减是结算依据
- 影响**工期**：变更可能改变关键线路
- 影响**责任划分**：变更原因决定费用由谁承担

因此，**变更原因分类**和**影响评估**是不能省略的硬性内容。

## 四、字段说明

### 变更原因分类的意义

变更原因决定了**费用由谁承担**：

| 原因 | 通常的费用承担方 |
|------|-----------------|
| A. 设计优化 | 视合同条款；常由设计单位承担超支或不增费 |
| B. 业主要求 | 建设单位 |
| C. 现场实际 | 视情况；勘察方 / 建设单位 / 设计单位之间分摊 |
| D. 规范更新 | 一般由建设单位承担 |
| E. 上位审查意见 | 通常由设计单位承担（设计原应符合规范） |

**SKILL 不要替用户判断费用承担方**——这是合同和商务的事，超出文档生成的职责。但可以在文档末尾提示"本变更费用承担方需依据合同另行确认"。

### 影响评估为什么不能空着

许多变更单的影响评估栏目被填成空白或写"无影响"敷衍过去。这会带来两个风险：

1. **结算阶段争议**——变更签字时未声明影响，事后追认困难
2. **设计责任**——若实际有影响而未评估，设计单位可能被认定未尽审慎义务

因此哪怕是"经评估，对工期和造价无影响"，也比留白好。如果用户没法量化（"现在还不知道造价影响多少"），写"暂无法量化，需另行算量"也是合规的。

### 涉及图纸的版次跟踪

变更后图纸**必须升版**。版次规则：

- 主要修改：升大版（Rev.A → Rev.B → Rev.C）
- 微小修订：升小版（Rev.B.1 → Rev.B.2，部分单位用此规则）
- 多数单位只用大版，遇修订即 +1

变更单中"原版次"指本次变更**前**已发出的版次；"新版次"指变更**后**新发的版次。SKILL 不能自己造版次号，必须问设计单位实际版次。

## 五、常见错误

1. **变更原因写"详见附件"或"如下"**——必须明确分类（A/B/C/D/E/F），单选。
2. **影响评估全填"无影响"但实际有影响**——签字方应核实，但 SKILL 在用户填全"无"时应提示"是否需要再确认一下？"
3. **"变更前/变更后"用同一张图截图**——必须能看出变化。如果是文字描述变更，前后段落要可对比。
4. **签字栏由 SKILL 替设计师签字**——绝对禁止。签字栏只留位置和姓名输入位（如必要），不要替任何人"代签"。
5. **遗漏对其他专业的影响**——暖通改了管路标高，往往影响电气桥架、给排水管线、装饰吊顶净高。SKILL 应当主动提示用户检查相关专业。
6. **变更生效日期早于签发日期**——除非是追溯性确认（这种情况要在原因里说明），否则生效日期应等于或晚于签发日期。
7. **附件引用业主指令但不附原件**——选 B（业主要求）必须附业主书面指令；选 E（审查意见）必须附审查意见原件或编号。

## 六、与其他文档的衔接

- 如果变更由 RFI 触发，变更单"原因详细说明"应引用 RFI 编号
- 如果变更涉及多个专业，应**配套出具其他专业的变更单**，编号顺延（见示例中暖通变更引发电气变更）
- 变更应同步更新项目的 **变更台账**——SKILL 生成变更单时建议提示用户登记台账
- 如变更涉及工程量增减，须由总包另行提交 **工程量签证** 或 **变更签证单**——这两份是不同文件，不要混淆
