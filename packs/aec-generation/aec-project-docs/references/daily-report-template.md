# 施工日报模板

> **何时读取此文件**：当需要生成施工日报、监理日志、施工日志、项目部日报时。

## 一、文档定位

施工日报是**法定档案**——施工过程中每日记录的主要凭证。在工程争议、安全事故、质量问题追溯时，日报往往是关键证据。

因此施工日报有几条不能动的硬性要求：

1. **每日一份**，不能合并几日补写
2. **不能空缺关键字段**——尤其是天气、安全/质量、问题处理
3. **当日填写当日事**——不能预填"明日计划"作为今日完成
4. **签字栏由真人签字**——不能由 SKILL 代签

SKILL 的职责是**协助整理**用户提供的素材，不是替项目编造日报。如果用户的素材不完整，明确指出缺什么，让用户补充，而不是自己脑补。

## 四、天气描述规范

天气是日报中**法定必填项**，因为它直接影响：

- 混凝土施工（高温、低温、降雨都有施工要求）
- 高空作业（风力 ≥ 6 级一般停止露天作业）
- 防水施工（降雨、潮湿环境不能进行）
- 工期顺延（恶劣天气是合同允许的工期顺延理由之一）

### 标准描述要素

| 要素 | 写法 |
|------|------|
| 天气状况 | 晴 / 多云 / 阴 / 雨 / 雪 / 雾 / 霾 / 雷阵雨 等 |
| 温度 | 最低–最高 ℃，如 "17–22℃" |
| 风力 | 风向 + 几级，如 "东北风 3–4 级" |
| 湿度 | 百分数（混凝土养护、防水施工需关注） |

### 转换天气节点

如当日天气有显著变化（晴转阴、雨起止），必须在"特殊气象"栏记录**转换的具体时间和持续时长**。

例：
- ✅ "14:30 起小雨，至 16:20 止；雨量约 2mm，未达停工阈值"
- ❌ "下午有雨"（无法判断对施工的实际影响）

### 影响施工的恶劣天气

以下情形必须在日报中显式记录，作为后续工期顺延或质量异议的依据：

- 降雨 ≥ 5mm 且持续超过 30 分钟
- 大风 ≥ 6 级
- 气温 < 0℃ 或 > 35℃
- 浓雾、霾导致能见度 < 200m
- 雷暴

如因天气**部分或全部停工**，需注明：

- 停工时段
- 停工范围（哪些工序、哪些工作面）
- 复工时间

## 五、字段说明

### 日报编号的特殊规则

施工日报通常**按日期编号**而非流水号，便于按日期检索：

```
{项目代号}-DR-YYYYMMDD
例：SHCS-DR-20251018
```

如一日有多份（不同分包各出一份），加分包代号：

```
SHCS-DR-20251018-八局土建
SHCS-DR-20251018-暖通分包
```

### 关于"无"的写法

若某栏确实无内容（如"安全事故 / 未遂事故"），写"无"——**不能空白**。空白会被视为漏填。

但 SKILL 必须警惕"全是无"的素材：

- "问题与处理：无" + "整改：无" + "事故：无" + "协调：无" 同时出现，要主动问用户："今天确实风平浪静吗？是否有些事情没记录下来？"
- 现场工程不可能完全无问题。 100% "无" 通常意味着没记录，不是没发生。

### 监理签认栏

监理签认是日报合规的关键。**SKILL 不能替监理"勾选已签"**。监理是否签认是事实问题，由用户告知或留空。

如用户告知"今日监理未到场"，应在日报中注明并由次日补签，而非伪造签认。

## 六、常见错误

1. **天气描述模糊**——"晴"、"阴"——没有温度、风力，无法判断对施工的影响。
2. **人材机数据没有班组划分**——总数 80 人没用，必须能追溯到哪个班组的哪个工种。
3. **工程量没有单位或没有计划对比**——只写"完成钢筋绑扎"无法判断进度。
4. **"问题与处理"栏目下没有时间和处理状态**——问题闭环管理依赖这两项。
5. **预填明日计划当作今日完成**——SKILL 在用户提供的素材里如果发现这种情况，必须指出。
6. **签字栏由 SKILL 代签**——绝对禁止。
7. **合并几日的日报**——日报必须每日一份。如果用户要补填多日，SKILL 应分别生成多份，而不是合成一份"近三日工作汇总"。
8. **使用预测性语言**——日报是事实记录，不是计划。"预计完成 90%" 应改为"截至 18:00 完成 60%，剩余 40% 转入次日"。

## 七、与其他文档的衔接

- 日报中的 RFI / 变更单引用 → 编号必须真实存在，不能造编号
- 日报积累为**周报**——SKILL 生成周报时可调用本周 7 份日报作为输入
- 安全事故记录 → 触发独立的事故报告流程，日报中只做提示性记录
- 监理日志格式与施工日报基本相同，但视角是监理方——主要变化在"填报单位"和"签字"栏，议题侧重于巡视、旁站、见证取样
