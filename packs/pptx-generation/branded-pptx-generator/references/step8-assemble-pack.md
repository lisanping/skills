# Step 8 — Assemble & Pack

Drop all template sample slides (they were references only), reorder
the new slides to match `slide-plan.json`, and rebuild section grouping.

```bash
# 1. Edit unpacked/ppt/presentation.xml → <p:sldIdLst>
#    Remove any <p:sldId> referencing original sample slides.
#    Reorder remaining entries to match slide-plan.json's `slides[]`.

# 2. Rebuild <p14:sectionLst> in presentation.xml
#    The template's section grouping references old sample slide IDs.
#    In <p:extLst>, find the <p:ext> with URI
#    "{521415D9-36F7-43E2-AB2F-B90AF26B5E84}" and replace its
#    <p14:sectionLst> entirely:
#      - One <p14:section> per section in content-outline.json.
#      - name = section title; id = fresh GUID.
#      - Each section's <p14:sldIdLst> lists the matching slide IDs
#        from the new <p:sldIdLst> (map via slide-plan.json taskIds).
#    If the template has no <p14:sectionLst>, skip this sub-step.

# 3. Drop orphaned files
python $PPTX_SKILL/scripts/clean.py unpacked/

# 4. Clean [Content_Types].xml
#    clean.py removes slide/media files but does NOT remove their
#    entries from [Content_Types].xml. Orphaned <Override> entries
#    cause PowerPoint to trigger a repair prompt.
#    Walk all <Override> elements, remove any whose PartName file
#    no longer exists in unpacked/.

# 5. Update docProps/app.xml
#    The template's app.xml still declares the original slide count
#    and title list. Update:
#      - <Slides> → new slide count
#      - <TitlesOfParts><vt:vector> → new slide titles
#    (Stale counts/titles don't trigger repair but look wrong in
#    file-manager previews.)

# 6. Compute and patch agenda page numbers
#    Now that <p:sldIdLst> is finalized (sample slides removed,
#    new slides reordered), walk it in order and compute each
#    content-outline.json section's starting slide number (1-based).
#    Find the agenda slide XML in unpacked/ppt/slides/, locate the
#    page-number placeholders (slots named agenda_page_N in
#    slide-content.json), and replace "TBD" text with actual numbers.
#    If no agenda slide exists in slide-plan.json, skip this sub-step.

# 7. Detach stale chart data sources
#    If any chart XML was edited by subagents (categories, values,
#    dimensions changed), the embedded .xlsx is now stale.
#    For each edited chart:
#      a. Remove <c:externalData> from chart XML
#      b. Remove the corresponding rId from chart .rels
#      c. Delete the .xlsx from ppt/embeddings/
#      d. Remove the .xlsx Override from [Content_Types].xml
#    This makes charts "cache-only" — they render correctly but
#    "Edit Data" in PowerPoint won't work. The alternative (also
#    updating xlsx contents) is complex and rarely needed.
#    WARNING: Skipping this step is a confirmed repair trigger.

# 8. Verify chart cache integrity
#    For every <c:numCache> and <c:strCache> in every <c:ser>,
#    verify <c:ptCount val="N"/> matches the actual number of
#    <c:pt> child elements. Subagents frequently change the number
#    of data points without updating ptCount.
#    WARNING: ptCount mismatch is a confirmed repair trigger.

# 9. Reject connector shapes (undetectable corruption source)
#    <p:cxnSp> elements generated by python-pptx lose implicit
#    context (stCxn/endCxn references) during spTree merge,
#    producing XML that passes all automated checks but causes
#    PowerPoint COM to reject the entire file.
#    This check MUST run before packing — it is the only automated
#    gate that catches this class of silent corruption.
if grep -rl "cxnSp" unpacked/ppt/slides/slide*.xml 2>/dev/null; then
  echo "BLOCKED: <p:cxnSp> connector shapes found."
  echo "Replace with <p:sp> + prstGeom prst=\"line\" (see XML Editing Rules)."
  echo "Offending files:"
  grep -rl "cxnSp" unpacked/ppt/slides/slide*.xml
  exit 1
fi

# 10. Pack with validation
python $PPTX_SKILL/scripts/office/pack.py unpacked/ output.pptx \
  --original $TEMPLATE_FILE
```

**Checkpoint:** `$SESSION/output.pptx` exists and is valid OOXML;
no `<p:cxnSp>` elements in any slide XML (sub-step 9 gate passed);
`<p14:sectionLst>` (if present) contains only new sections matching
`content-outline.json`, with no stale slide IDs from the template;
no orphaned `[Content_Types].xml` entries; no stale chart `.xlsx` files;
agenda page numbers patched with final values (if agenda slide exists).
