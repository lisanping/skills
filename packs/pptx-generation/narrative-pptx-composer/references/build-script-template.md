# s07-build.py — Script Template

Used in Step 7a. The `s07-build.py` script consumes the Step 5–6
artifacts and emits a `.pptx` via python-pptx.

## Generation approach

Unlike a generic zone-loop renderer, `s07-build.py` uses **per-slide
builder functions** generated by the LLM for each composition type.
This gives full python-pptx control over compound elements (cards
with badges, overlays, accent bars, rounded rects with text) that
a flat zone→API mapping cannot express.

The shared helpers library (`$COMPOSER_SKILL/scripts/s07_slide_helpers.py`)
provides reusable building blocks. The LLM imports these and
composes per-slide builder functions that read slot data from
`s06-slide-content.json`.

## Critical python-pptx rules

See [python-pptx-guide.md § Critical Rules](python-pptx-guide.md)
for the full list (RGBColor, Inches/Pt, MSO_SHAPE, word_wrap,
border removal, blank layout, z-order). **Read that section before
writing `s07-build.py`.**

**Corner radius reminder.** Use `corner_radius_in` (absolute inches)
on `add_rounded_rect` and `add_card`, never bare `adjustments[0]`
(a fraction of the shorter side — produces inconsistent visuals
across small and large cards). See the helper docstrings in
`s07_slide_helpers.py` for the conversion math and the recommended
radius table.

## Image handling rules (mandatory)

- **Image path resolution — use the helper, not an inline expression.**
  After Step 5f patches, the resolved image path lives on the **plan
  zone** (`s05-slide-visual-design.json → slides[i].layoutSpec.zones[j].path`),
  NOT on the content slot. `s06-slide-content.json` slot entries for
  image zones may carry `"path": null` — that is by design.

  Always resolve via `resolve_image_path(plan_zone, slot)` from
  `s07_slide_helpers.py`:

  ```python
  from s07_slide_helpers import resolve_image_path

  img_path = resolve_image_path(plan_zone, slot)
  add_image_safe(slide, img_path, left, top, width, height)
  ```

  Reading only from the slot silently renders empty placeholders for
  every generated illustration. This bug class is invisible to
  text-based validators (`s06_validate_content.py` only checks slot
  presence, not path resolution) — Step 8 QA Round 1's visual
  subagent is the only catcher, and only after the deck is fully
  rendered. Routing every image-zone path through `resolve_image_path`
  removes this entire bug class from the prompt-memory budget.
- **Preserve original aspect ratio:** Use the `add_image_safe()`
  helper which computes display dimensions to fit within the zone
  while maintaining the original width:height ratio. If the image
  file is missing, it renders a placeholder rectangle instead of
  breaking the build.
- **No image reuse:** Each image file may appear on at most one
  slide. Track assigned images across the deck and enforce uniqueness.
  If a duplicate is detected, remove the image from the lower-priority
  slide and use a text-only layout instead.

## What the script does

1. Creates a new `Presentation()` and sets slide dimensions.
2. Loads `s06-slide-content.json` and `s05b-style-policy.json`.
3. Defines per-slide builder functions (e.g. `build_s01`,
   `build_diagram_callouts`, `build_card_grid`) that use
   helpers from `s07_slide_helpers.py`.
4. For each slide in `s06-slide-content.json` order:
   a. Calls the appropriate builder function.
   b. The builder creates a slide, sets background, adds shapes
      and text boxes with full compositional control.
   c. Adds speaker notes via `notes_slide.notes_text_frame`.
5. Calls `prs.save()`.

## Per-slide builder pattern

### Builder function conventions (mandatory)

Three styles have appeared in past sessions; only the first two are
allowed.

| Style                                                                                                          | Status         | When                                                                                                                                                                                             |
| -------------------------------------------------------------------------------------------------------------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Per-`taskId` builder** — one `build_<taskId>` per slide                                                      | **Default**    | Slide is unique, or its composition is not reused.                                                                                                                                               |
| **Shared composition builder** — one `build_<composition>` reused by multiple `taskId`s via the dispatch table | Allowed        | Two or more slides share an identical layout pattern (e.g. four card-grid body slides). Map each `taskId` to the shared function.                                                                |
| **Generic dispatch** — a single `build_slide(tid)` that branches on `layoutPattern` internally                 | **Disallowed** | Pushes composition logic into a switch statement, defeats the per-slide builder rationale (compound elements like card+badge+overlay+accent-bar cannot be expressed as a flat zone→API mapping). |

**Rule.** The dispatch unit is always the `taskId`. The builder body
is owned per-slide by default; promote a body to a shared
`build_<composition>` only when concrete reuse exists. Do not write
builders speculatively for compositions that appear once.

```python
builders = {
    "s01": build_s01,                # unique cover
    "s02": build_s02,                # unique agenda
    "s03": build_card_grid,          # shared
    "s04": build_card_grid,          # shared
    "s05": build_card_grid,          # shared
    "s06": build_diagram_callouts,   # shared
    "s07": build_diagram_callouts,   # shared
    "s08": build_s08,                # unique closing
}
```

When a shared builder needs per-slide variation, parameterise it via
function arguments (e.g. `build_diagram_callouts(sd, is_hero=False)`)
rather than branching on `taskId` inside.

### Common patterns

The LLM reads the `layoutPattern` from `s05-slide-visual-design.json`
and the `layoutSpec` zone coordinates, then writes the builder.

### Bookend slides (cover, closing, divider)

Built from the same `layoutSpec` as body slides, in the same
imperative per-slide-builder style. Read the `background` and each
`zone` from `s05-slide-visual-design.json`, then emit direct python-pptx
calls (and helpers) for each element. Apply the background **first**
(see [python-pptx-guide.md § Critical Rules #6](python-pptx-guide.md)
for the canonical z-order rule).

```python
def build_cover(sd):
    slide = prs.slides.add_slide(blank_layout)

    # 1. Background — solid color or full-bleed image.
    #    For image-typed backgrounds ALWAYS use set_slide_bg_image
    #    (it covers the full canvas at slide_w × slide_h, NOT the
    #    bounds of any zone). Pass scrim_color/scrim_alpha when the
    #    overlaid text needs legibility help — decide per slide based
    #    on text color, image luminance, and visual register.
    bg = sd["layoutSpec"].get("background", {})
    if bg.get("type") in ("image", "image-generated"):
        scrim = bg.get("scrim", {})  # optional: {color, alpha}
        set_slide_bg_image(slide, bg.get("path"),
                           fit=bg.get("fit", "cover"),
                           fallback_color=PAL.get("primary"),
                           scrim_color=resolve_color(scrim["color"])
                                       if scrim.get("color") else None,
                           scrim_alpha=scrim.get("alpha", 0.0))
    else:
        set_slide_bg(slide, resolve_color(bg.get("color", PAL["primary"])))

    # 2. Title — read slot text, position from the title zone.
    title_zone = next(z for z in sd["layoutSpec"]["zones"]
                      if z["role"] == "title")
    title_slot = get_slot(sd["slots"], "title")
    add_textbox(slide,
                Inches(title_zone["x"]), Inches(title_zone["y"]),
                Inches(title_zone["w"]), Inches(title_zone["h"]),
                title_slot["text"], font_size=44, bold=True,
                color=PAL.get("text-on-primary", RGBColor(0xFF, 0xFF, 0xFF)))

    # 3. Optional subtitle, decoration accents, etc. — emit per zone.
    add_accent_bar(slide, Inches(0.6), Inches(4.2),
                   width=Inches(1.5), color=PAL["accent"])

    add_speaker_notes(slide, sd.get("speakerNotes", ""))
```

**Full-bleed background rule (mandatory).** Cover, closing, and
divider slides whose intent is "image fills the canvas" MUST express
the image as `layoutSpec.background` with `type: "image-generated"`
and render via `set_slide_bg_image()`. Do NOT model a near-full-canvas
image as a `hero-image` zone and then call `add_image_with_overlay()`
for it — that path obeys zone bounds, so any aspect-ratio mismatch or
sub-canvas zone produces letterboxing or a centered-but-not-bleeding
picture. `add_image_with_overlay()` is reserved for **partial-canvas**
photo zones (e.g. a half-slide hero alongside a text column).

**Scrim is decided per slide, not per deck.** Default to no scrim.
Add one only when needed for legibility — pick color and alpha from:

| Situation                                                      | Scrim                                      |
| -------------------------------------------------------------- | ------------------------------------------ |
| Light text over busy / mid-luminance image                     | `scrim_color=black, scrim_alpha=0.35–0.55` |
| Dark text over light image                                     | `scrim_color=white, scrim_alpha=0.35–0.50` |
| Editorial cover, bold display text, image already low-contrast | omit scrim (alpha=0)                       |
| Restrained register, instructional / regulatory                | minimal scrim (alpha ≤ 0.25)               |

For a partial-canvas hero zone (image occupies a defined region with
text alongside), use `add_image_with_overlay` on that zone:

```python
hero = next(z for z in sd["layoutSpec"]["zones"] if z["role"] == "hero-image")
add_image_with_overlay(slide,
    hero.get("path"),
    Inches(hero["x"]), Inches(hero["y"]),
    Inches(hero["w"]), Inches(hero["h"]),
    scrim_color=RGBColor(0, 0, 0), scrim_alpha=0.45,
    fallback_color=PAL["primary"])
# Title is added *after* so it paints above the scrim.
add_textbox(slide, ..., title_slot["text"], ...)
```

### Card grids (mosaic-2x3, four-card-grid, etc.)

Loop over the cards and call `add_card()` per cell — the helper
emits the rounded rect, optional badge, title, and body in one
call. Read each card from `get_slot(sd["slots"], f"card-{i+1}")`,
compute `(x, y)` from `(col, row)` + grid gaps, and pass
`accent_color=PAL["accent"]`, `corner_radius_in=0.12`. Omit
the accent bar (card backgrounds already provide structure).

### Diagram with callouts

Left zone: image via `add_image_safe(slide, resolve_image_path(plan_zone, slot), ...)`.
Right zone: numbered callout badges — one `add_outline_circle()`
per callout for the badge ring + an `add_textbox()` with the numeral
centered, then a second `add_textbox()` for the callout body to the
right. Apply the title row first; render the diagram before the
callout text (z-order).

### Formula zones (`type: "formula"`)

Phase 1 (per `sessions/formula-svg-design-2026-05-01.md`) treats each
formula as its own zone; the renderer writes a **PNG @ 300dpi** under
`<session>/formulas/<taskId>-<role>.png` and embeds it as a picture.
Author the formula in `s06-slide-content.json`'s slot:

```jsonc
{
  "zone": "formula-attention",
  "formulaSource": "\\mathrm{Attention}(Q,K,V) = \\mathrm{softmax}\\!\\left(\\frac{QK^{\\top}}{\\sqrt{d_k}}\\right)V",
  "alt": "Attention of Q, K, V equals softmax of Q-K-transpose over square-root of d_k, applied to V."
}
```

…with the matching plan zone:

```jsonc
{
  "role": "formula-attention",
  "type": "formula",
  "x": 1.5, "y": 2.7, "w": 10.5, "h": 2.4,
  "notation": "mathtext",
  "source": "authored",
  "fitMode": "scale-to-zone",
  "alignment": "center",
  "verticalAlign": "middle",
  "colorToken": "ink",
  "fontPt": 36,
  "fallback": { "kind": "text", "text": "Attention(Q,K,V) = softmax(QKᵀ/√d_k) V" }
}
```

**Build pattern** — call `render_formulas()` once at the top of
`main()` (before any slide is built) so every formula is rendered
and cached, then per-slide builders just look up the cached path:

```python
# ── Pre-pass: render every formula PNG once and cache ────────────
formula_log: list[dict] = []
formula_cache = render_formulas(
    plan,                          # parsed s05-slide-visual-design.json
    {"slides": slide_data},        # parsed s06-slide-content.json
    session_dir=SESSION,
    style_policy=style,            # parsed s05b-style-policy.json
    build_log=formula_log,
)
for entry in formula_log:
    print(f"  formula {entry['taskId']}/{entry['role']}: {entry['action']}")

# ── Per-slide builder ────────────────────────────────────────────
def build_s05_attention(sd):
    """Hero formula slide with title + lead + caption."""
    slide = prs.slides.add_slide(blank_layout)
    set_slide_bg(slide, PAL["background"])

    # Title
    title_slot = get_slot(sd["slots"], "title")
    add_textbox(slide, Inches(0.5), Inches(0.5), Inches(12.5), Inches(1.0),
                title_slot["text"], font_size=36, bold=True, color=PAL["ink"],
                font_name=style["typography"]["headingFont"], alignment=PP_ALIGN.CENTER)

    # Lead
    lead_slot = get_slot(sd["slots"], "lead")
    add_textbox(slide, Inches(1.5), Inches(1.7), Inches(10.5), Inches(0.7),
                lead_slot["text"], font_size=20, color=PAL.get("secondary", PAL["ink"]),
                alignment=PP_ALIGN.CENTER)

    # Formula — locate the plan zone, look up the cached artifact,
    # call add_formula with the cached path.
    plan_slide = next(s for s in plan["slides"] if s["taskId"] == sd["taskId"])
    fz = next(z for z in plan_slide["layoutSpec"]["zones"] if z["role"] == "formula-attention")
    fslot = get_slot(sd["slots"], "formula-attention")
    add_formula(
        slide,
        fslot["formulaSource"],
        Inches(fz["x"]), Inches(fz["y"]), Inches(fz["w"]), Inches(fz["h"]),
        color=PAL.get(fz.get("colorToken", "ink"), PAL.get("ink", RGBColor(0x22, 0x22, 0x22))),
        font_pt=fz.get("fontPt", 36),
        fontset=style.get("typography", {}).get("mathFontset", "cm"),
        alignment=fz.get("alignment", "center"),
        vertical_align=fz.get("verticalAlign", "middle"),
        fit_mode=fz.get("fitMode", "scale-to-zone"),
        cache_path=formula_cache.get((sd["taskId"], "formula-attention")),
        fallback_text=fz.get("fallback", {}).get("text", ""),
        alt=fslot.get("alt"),
    )

    # Caption
    cap_slot = get_slot(sd["slots"], "caption")
    if cap_slot:
        add_textbox(slide, Inches(1.5), Inches(5.4), Inches(10.5), Inches(0.6),
                    cap_slot["text"], font_size=14, color=PAL.get("textMuted", PAL["ink"]),
                    alignment=PP_ALIGN.CENTER)

    add_speaker_notes(slide, sd.get("speakerNotes", ""))
```

**Notes:**

- `render_formulas` is **idempotent** via a `.png.sig` sidecar;
  re-running the build only re-renders formulas whose source / color /
  fontset / fontPt actually changed.
- The helper accepts the tuple `(taskId, role)` as the cache lookup
  key — pass the matching `cache_path` to `add_formula` so the
  per-slide builder skips a redundant render.
- `fontset` is read from `style.typography.mathFontset` (default
  `"cm"` — Computer Modern, the LaTeX-faithful classical math fontset);
  validate values via `s05_validate_plan.py`.
- When `colorToken` is omitted, the helper inherits body color via
  `palette.body → palette.ink → #222222`. Override only for hero
  formulas (`colorToken: "accent"`).
- Forbidden: don't author `$…$` or `\(...\)` delimiters in
  `formulaSource` — `s06_validate_content.py` hard-fails on those
  because the renderer wraps the body itself.

#### Authoring rules

| Rule                                                           | Detail                                                                                                                                                                                                                                                                                                         |
| -------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use `formula` zones for typeset math**                       | Don't fake fractions, integrals, or sub/superscripts in textboxes (`a/b`, `x^2`, `sum(...)`). The `formula` zone renders true mathtext glyphs at any zoom. Author the source in `s06-slide-content.json`'s slot as `formulaSource` (mathtext, no `$…$` delimiters) and let `add_formula()` do the typesetting. |
| **`alt` is mandatory on every formula slot**                   | Equation slides without alt are a P0 a11y bug. Write a plain-language read-out (≥ 10 chars, no LaTeX commands) — `s06_validate_content.py` hard-fails when missing.                                                                                                                                            |
| **No `$…$` / `\(…\)` / `\[…\]` delimiters in `formulaSource`** | The renderer wraps with `$…$` itself; doubled delimiters silently produce empty artifacts.                                                                                                                                                                                                                     |
| **One formula per zone**                                       | Inline math is out of scope. Each formula occupies its own `type: "formula"` zone — author the prose around it in adjacent text zones.                                                                                                                                                                         |
| **Mathtext-only notation; no LaTeX packages**                  | Supports matplotlib mathtext only (fractions, roots, sub/sup, Greek, sums, integrals, common operators). **No `\begin{matrix}`, `amsmath`, `bm`, custom commands.** Use stacked sup or `\substack` for compact alternatives.                                                                                   |
| **Don't insert SVG via `add_picture()`**                       | python-pptx's `add_picture` routes through Pillow, which does not understand SVG. `add_formula()` already renders PNG @ 300dpi and inserts it correctly — call the helper, not the underlying API.                                                                                                             |

## Text fitting — owned by Step 6

Step 7 makes **no fitting decisions**. The two-stage trimming in
Step 6b is the single owner of "will this text fit this zone at this
font size"; by the time Step 7 runs, slot text and font-size tokens
in `s06-slide-content.json` are guaranteed to fit.

**Always use `add_textbox`.** Do not call `add_textbox_autofit`
speculatively at Step 7. A runtime font shrink at Step 7 is a second
fitting decision that masks Step 6b heuristic errors instead of fixing
them. The helper still exists in `s07_slide_helpers.py` as an escape
hatch; if you find yourself reaching for it, treat that as a Step 6
bug to log in `s09-session-retrospective.json` (so the 6b heuristic
can be tuned), not as a routine Step 7 tool.

## Step 8 QA override scaffolds (per-slide and per-cohort)

When Step 8 fix cycles hand back remediations, the designer LLM adds
or extends two override dicts at the top of `s07-build.py`. These
dicts are consulted **before** any token resolution from
`s05-slide-visual-design.json`, so the design-intent JSON stays
unchanged and the next QA round can still reason about original tokens.

### Per-slide overrides (`PERSLIDE_*_OVERRIDES`)

For single-page fixes scoped to one `(taskId, role)` pair. Naming
convention: one dict per attribute family (`COLOR`, `FONT_PT`,
`BBOX`). Example for color:

```python
PERSLIDE_COLOR_OVERRIDES = {
    ("s01", "subtitle"): "#94A3B8",   # i-s01-001 secondary→slate-400
    ("s12", "card-1-tab-label"): "#B45309",
}
```

The resolver looks up the override; falls back to the design token.

### Per-cohort overrides (`COHORT_OVERRIDES`)

Step 8d (per-cohort QA) emits remediations that apply to **every member
of a cohort uniformly**. Encoding them as N per-slide entries is fragile
(easy to miss a member → cohort verify keeps failing). Use a single
`COHORT_OVERRIDES` dict keyed by `(cohortId, role)` plus a static
`COHORT_TO_MEMBERS` map (copied verbatim from
`s08-cohort-definitions.json`):

```python
# ── Step 8d cohort overrides (added by per-cohort QA fix cycle) ──────
# Source: s08-cohort-remediations.json + s08-cohort-definitions.json

COHORT_TO_MEMBERS = {
    "act-dividers": ["s03", "s06", "s10", "s16"],
    "bookends":     ["s01", "s23"],
    # ... mirror s08-cohort-definitions.json cohorts[].members exactly
}

# (cohortId, role) → partial override of {fontPt, fontFamily, color,
#                    bold, w, h, x, y, fill}. Only listed attributes
#                    are overridden; others keep design-JSON values.
COHORT_OVERRIDES = {
    # c-act-dividers-001 — '02'/'03'/'04' wrap at 280pt; shrink across
    # all 4 dividers so the family stays single-line.
    ("act-dividers", "act-tag"): {"fontPt": 220},
}

# Pre-expand to a flat (taskId, role) -> patch dict at module load.
PERSLIDE_FROM_COHORT_OVERRIDES = {}
for (_cid, _role), _patch in COHORT_OVERRIDES.items():
    for _tid in COHORT_TO_MEMBERS.get(_cid, ()):
        PERSLIDE_FROM_COHORT_OVERRIDES.setdefault((_tid, _role), {}).update(_patch)
```

In zone-rendering helpers, **per-slide overrides take precedence over
cohort overrides** (per-slide is more specific):

```python
def resolve_zone_font_pt(tid, zone):
    role = zone.get("role")
    # Per-slide first.
    ps = PERSLIDE_FONT_PT_OVERRIDES.get((tid, role))  # if dict exists
    if ps is not None:
        return int(ps)
    # Then cohort.
    co = PERSLIDE_FROM_COHORT_OVERRIDES.get((tid, role), {}).get("fontPt")
    if co is not None:
        return int(co)
    # Fall back to design token.
    return font_size_pt(zone.get("fontSize"))
```

When a Step 8d fix targets only **some** members of a cohort (not the
whole family), encode it as per-slide overrides instead — the cohort
dict is reserved for cases where uniformity is the design contract.

## Helpers for technical compositions

When `s05b-style-policy.json → visualTone.contentDomain` is
`technical-instructional`, the technical layouts depend on a few
specific helpers and conventions. Cross-references:
[layout-principles.md](layout-principles.md) for the layout
catalog and [step5d-slide-layout.md](step5d-slide-layout.md) for selection.

| Need                                                | Use                                                                                                                                                      |
| --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Resolve monospace font                              | `resolve_font("mono", style["typography"])` — returns `typography.monoFont` or `"JetBrains Mono"` fallback. Code zones must NOT fall back to `bodyFont`. |
| Step-flow / before-after arrow                      | Default `"→"` (U+2192) when the slot is missing, else inject `"⟶"` (U+27F6) from the slot. Do not skip — the visual flow breaks silently.                |
| `architecture-layered` labels                       | Read `layer-N-label` slots; warn (do not error) on empty.                                                                                                |
| Code blocks (`code-explain-split`, `terminal-hero`) | Pass raw source with `\n` preserved straight to `add_textbox` using the monospace font. The helper handles newline splitting.                            |

### Callout numbers (`diagram-with-callouts`)

The `callout-N-num-text` zones display numerals inside

write text into its `text_frame`. Create the circle before the
numeral textbox — see
[python-pptx-guide.md § Critical Rules #6](python-pptx-guide.md)
